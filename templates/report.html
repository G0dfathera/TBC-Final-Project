How to Report a Vulnerability

1. Provide Vulnerability Details

Clearly describe the vulnerability, including the type (e.g., SQL Injection, XSS), and where it was found. Provide steps to reproduce the issue.

2. Impact Assessment

Describe the potential impact if the vulnerability is exploited. For example, unauthorized data access, privilege escalation, etc.

3. Mitigation Suggestions

If possible, include suggestions on how to fix the vulnerability. For example, "Use parameterized queries to prevent SQL injection." This can expedite the resolution process.

Sample Vulnerability Report

Field Description
Vulnerability Type SQL Injection
Details The login form at /login does not properly sanitize user input, allowing attackers to inject malicious SQL code and bypass authentication.
Impact Allows unauthorized access to the system and potential data exposure.
Suggested Fix Use prepared statements or parameterized queries to prevent SQL injection.
